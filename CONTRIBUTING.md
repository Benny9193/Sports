# Contributing to Sports Compendium

Thank you for your interest in contributing to the Sports Compendium! This document provides guidelines for contributing to this project.

## How to Contribute

### Adding New Sports

1. **Fork the repository** and create a new branch for your changes
2. **Update the data** in `app.js`:
   - Locate the appropriate category in the `sportsData` object
   - Add your sport with a name and description
   - Follow the existing format:
     ```javascript
     { name: "Sport Name", description: "Brief description of the sport" }
     ```
3. **Keep descriptions concise** - Aim for 1-2 sentences
4. **Ensure accuracy** - Verify your information from reliable sources
5. **Submit a pull request** with a clear description of your changes

### Improving Existing Content

- **Corrections**: If you find errors, please open an issue or submit a PR with the correction
- **Enhanced descriptions**: Feel free to improve existing sport descriptions
- **Categorization**: Suggest better categorization if a sport seems misplaced

### Code Contributions

#### Bug Fixes
- Open an issue describing the bug
- Submit a PR referencing the issue
- Include steps to reproduce if applicable

#### New Features
- Open an issue to discuss the feature first
- Get feedback before implementing
- Follow existing code style and conventions
- Update documentation as needed

### Style Guidelines

#### JavaScript
- Use ES6+ features
- Keep functions focused and single-purpose
- Add comments for complex logic
- Use meaningful variable names

#### CSS
- Follow the existing CSS variable naming convention
- Keep selectors specific but not overly complex
- Ensure responsive design principles
- Test on multiple screen sizes

#### HTML
- Use semantic HTML5 elements
- Maintain accessibility standards
- Keep markup clean and well-indented

## Code of Conduct

### Our Standards

- Be respectful and inclusive
- Welcome newcomers and help them learn
- Focus on constructive feedback
- Accept responsibility for mistakes
- Prioritize what's best for the community

### Unacceptable Behavior

- Harassment or discriminatory language
- Trolling or insulting comments
- Publishing others' private information
- Any conduct that would be inappropriate in a professional setting

## Questions?

Feel free to open an issue for:
- Questions about contributing
- Clarification on guidelines
- Suggestions for improvement
- General feedback
